## 核心组件

### 1. RpcServer (Main Thread)

主线程负责监听和分发连接，不处理业务逻辑。

```
┌─────────────────────────────────────────────────────────────────────┐
│                        RpcServer::Start()                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   socket() ──> bind() ──> listen() ──> epoll_create1()             │
│                                                                     │
│   ┌───────────────────────────────────────────────────────────────┐ │
│   │                     主循环                                     │ │
│   │                                                               │ │
│   │   while (true) {                                              │ │
│   │       epoll_wait(listen_fd);                                  │ │
│   │                                                               │ │
│   │       while ((client_fd = accept4()) > 0) {                   │ │
│   │           peer_desc = FormatIPPort(client_addr);              │ │
│   │           worker = SelectWorker();  // 轮询选择               │ │
│   │           worker.EnqueueConnection(client_fd, peer_desc);     │ │
│   │       }                                                       │ │
│   │   }                                                           │ │
│   └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2. WorkerLoop (Worker Thread)

每个 WorkerLoop 在独立线程中运行，负责驱动多个连接的协程。

```
┌─────────────────────────────────────────────────────────────────────┐
│                        WorkerLoop 生命周期                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   Init()                                                            │
│     ├── epoll_create1()     → epoll_fd_                            │
│     ├── eventfd()           → wake_fd_ (跨线程唤醒)                 │
│     └── epoll_ctl(ADD)      → 注册 wake_fd_ 到 epoll               │
│                                                                     │
│   Start()                                                           │
│     └── std::thread(Run)    → 启动工作线程                         │
│                                                                     │
│   Run() [工作线程]                                                  │
│     └── while (!stop_requested_) {                                 │
│            PollOnce();                                              │
│         }                                                           │
│                                                                     │
│   RequestStop()                                                     │
│     ├── stop_requested_ = true                                     │
│     └── Wakeup()            → 唤醒 epoll_wait                      │
│                                                                     │
│   Join()                                                            │
│     └── thread_.join()      → 等待线程结束                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 3. PollOnce() 事件循环

```
┌─────────────────────────────────────────────────────────────────────┐
│                      PollOnce() 单次轮询                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   步骤1: epoll_wait(timeout)                                        │
│          │                                                          │
│          ├── 返回 0  (超时)                                         │
│          ├── 返回 >0 (有事件)                                       │
│          └── 返回 <0 (错误/EINTR)                                   │
│                                                                     │
│   步骤2: 检查所有连接的超时                                          │
│          for (connection in connections_) {                         │
│              connection.Tick(now);  // 检查读/写超时                │
│          }                                                          │
│                                                                     │
│   步骤3: 处理就绪的 I/O 事件                                         │
│          for (event in events) {                                    │
│              if (event.fd == wake_fd_) {                            │
│                  接管 pending_connections_                          │
│                  read(wake_fd_);  // 清除 eventfd                   │
│                  continue;                                          │
│              }                                                      │
│                                                                     │
│              if (EPOLLIN)    NotifyReadable();                      │
│              if (EPOLLOUT)   NotifyWritable();                      │
│              if (EPOLLHUP/ERR) CloseConnection();                   │
│          }                                                          │
│                                                                     │
│   步骤4: 检查协程状态，清理已结束的连接                              │
│                                                                     │
│   步骤5: 动态更新 epoll 事件注册                                     │
│          if (HasPendingWrite()) events |= EPOLLOUT;                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 新连接接管流程

┌─────────────────────────────────────────────────────────────────────────────┐
│                        新连接接管流程 (跨线程安全)                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Main Thread                                    Worker Thread              │
│       │                                               │                     │
│       │  accept4()                                    │                     │
│       │      │                                        │                     │
│       │      ▼                                        │                     │

│       │      │                                        │                     │
│       │      ▼                                        │                     │
│       │  EnqueueConnection(fd, peer_desc)            │                     │
│       │      │                                        │                     │
│       │      ├── lock(pending_mutex_)                │                     │
│       │      ├── pending_connections_.push()         │                     │

## co_await 与协程挂起恢复

### ReadRequestCo 的 awaiter 流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    co_await ReadRequestCo() 流程                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   协程执行 co_await connection->ReadRequestCo(&error)                       │
│       │                                                                     │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │ await_ready()                                                       │   │
│   │     ├── 返回 true  → 数据已就绪，不挂起                             │   │
│   │     └── 返回 false → 需要等待，执行 await_suspend                   │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│       │                                                                     │
│       │ (如果需要等待)                                                      │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │ await_suspend(handle)                                                │   │
│   │     ├── read_waiter_ = handle      // 保存协程句柄                  │   │
│   │     ├── ArmReadDeadline()          // 设置超时定时器                │   │
│   │     └── return true                // 挂起协程                      │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│       │                                                                     │
│       │ (协程挂起，控制权返回给事件循环)                                    │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │ PollOnce() 继续执行                                                 │   │
│   │     epoll_wait() 等待事件                                           │   │
│   │     ...                                                             │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│       │                                                                     │
│       │ (socket 变为可读)                                                   │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │ NotifyReadable()                                                    │   │
│   │     └── ResumeWaiter(read_waiter_)  // 恢复协程                     │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│       │                                                                     │
│       ▼                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │ await_resume()                                                      │   │
│   │     ClearReadDeadline()  // 清除超时定时器                          │   │
│   │     return TryReadAndProcess()  // 读取数据并处理                   │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│       │                                                                     │
│       ▼                                                                     │
│   协程继续执行...                                                           │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 线程安全设计

### 跨线程通信

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        跨线程通信机制                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ❌ 错误做法：主线程直接操作 Worker 的 epoll                               │
│                                                                             │
│   Main Thread                     Worker Thread                            │
│       │                                │                                   │
│       ├── epoll_ctl(worker.epoll) ───>│  epoll_wait()  ← 数据竞争！        │
│       │                                │                                   │
│                                                                             │
│   ✅ 正确做法：投递队列 + eventfd 唤醒                                      │
│                                                                             │
│   Main Thread                     Worker Thread                            │
│       │                                │                                   │
│       ├── lock(pending_mutex_)        │                                   │
│       ├── pending_queue.push(fd)      │                                   │
│       ├── unlock                      │                                   │
│       │                                │                                   │
│       ├── write(wake_fd_, 1) ────────>│  epoll_wait() 返回                │
│       │                                │                                   │
│       │                                ├── read(wake_fd_)                  │
│       │                                ├── lock(pending_mutex_)            │
│       │                                ├── 取出 pending_queue              │
│       │                                ├── unlock                          │
│       │                                ├── epoll_ctl(ADD, fd)  ← 安全！    │
│       │                                └── 启动协程                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 线程归属检查

```cpp
bool IsOnOwnerThread() const noexcept {
  if (owner_thread_id_ == std::thread::id{}) {
    return true;  // 未初始化，默认允许
  }
  return owner_thread_id_ == std::this_thread::get_id();
}
```

用于确保某些操作只在所有者线程执行，防止跨线程误用。

## 性能特点

1. **one-loop-per-thread**：每个 WorkerLoop 独占一个线程和 epoll 实例
2. **无锁 I/O 处理**：连接一旦被 Worker 接管，所有 I/O 操作都在同一线程
3. **协程轻量级**：每个连接一个协程，比线程更轻量
4. **跨线程安全**：通过队列 + eventfd 实现安全的连接分发

## 配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| worker_count | Worker 线程数量 | 1 |
| read_timeout | 读超时时间 | 可配置 |
| write_timeout | 写超时时间 | 可配置 |
| epoll_wait_timeout | epoll 等待超时 | 100ms |
