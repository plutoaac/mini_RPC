## 2. 核心改进点 1: 主线程与工作线程分离

### 改进前

```
+-----------------------------------------------------------------------------+
|                           改进前 - 单线程模型                                |
+-----------------------------------------------------------------------------+
|                                                                             |
|   +---------------------------------------------------------------+        |
|   |                        Main Thread                             |        |
|   |                                                               |        |
|   |    +----------+    +----------+    +----------+    +----------+|        |
|   |    |  accept  |--->|  recv    |--->|  process |--->|  send    ||        |
|   |    |  新连接  |    |  阻塞等待|    |  处理请求|    |  阻塞发送||        |
|   |    +----------+    +----------+    +----------+    +----------+|        |
|   |                                                               |        |
|   |    问题：                                                     |        |
|   |    - accept 和请求处理在同一线程                              |        |
|   |    - recv/send 阻塞，无法处理其他连接                         |        |
|   |    - 连接多了就需要多线程，线程开销大                         |        |
|   |                                                               |        |
|   +---------------------------------------------------------------+        |
|                                                                             |
|   连接队列：                                                                 |
|   +---------+ +---------+ +---------+                                      |
|   | Conn 1  | | Conn 2  | | Conn 3  |  ... 串行等待处理                   |
|   | 处理中  | | 等待... | | 等待... |                                      |
|   +---------+ +---------+ +---------+                                      |
|                                                                             |
+-----------------------------------------------------------------------------+
```

|   |                  |  |                  |  |                  |        |
|   +------------------+  +------------------+  +------------------+        |
|                                                                             |
|   优点：                                                                     |
|   - 主线程专注于 accept，不被业务阻塞                                        |
|   - Worker 线程独立运行事件循环                                             |
|   - 连接被 Worker 接管后，所有 I/O 在同一线程完成                            |
|   - 无需加锁处理连接的 I/O 操作                                             |

## 3. 核心改进点 2: 协程驱动的异步 I/O

### 改进前：阻塞式 I/O

```
+-----------------------------------------------------------------------------+
|                          改进前 - 阻塞式 I/O                                 |
+-----------------------------------------------------------------------------+
|                                                                             |
|   线程执行流程：                                                             |
|                                                                             |
|   +----------+    +----------+    +----------+    +----------+             |
|   | recv()   |--->|  阻塞    |--->| 数据到达 |--->| 处理请求 |             |
|   | 调用     |    | 等待...  |    | 返回     |    |          |             |
|   +----------+    +----------+    +----------+    +----------+             |
|                          |                                                  |
|                          v                                                  |
|                   +----------------------------------------------+          |
|                   |  线程被阻塞，无法处理其他连接                  |          |
|                   |  CPU 空闲等待 I/O                              |          |
|                   |  资源浪费                                      |          |
|                   +----------------------------------------------+          |
|                                                                             |
|   问题：                                                                     |
|   - 一个线程只能处理一个连接                                                |
|   - I/O 等待时 CPU 空转                                                     |
|   - 大量连接需要大量线程                                                    |
|   - 线程切换开销大                                                          |
|                                                                             |
+-----------------------------------------------------------------------------+
```

### 改进后：协程异步 I/O

```
+-----------------------------------------------------------------------------+

+-----------------------------------------------------------------------------+
|                                                                             |
|   协程执行流程：                                                             |
|                                                                             |
|   +------------------+    +------------------+    +------------------+     |
|   | co_await         |    | 协程挂起         |    | 协程恢复         |     |
|   | ReadRequestCo()  |--->| 注册 EPOLLIN     |--->| 处理请求         |     |
|   +------------------+    +------------------+    +------------------+     |
|                                 |                                           |
|                                 v                                           |
|                    +----------------------------------------------+         |
|                    |  协程挂起期间：                                |         |
|                    |  - 线程继续运行事件循环                       |         |
|                    |  - 可以处理其他连接的 I/O                     |         |
|                    |  - 单线程可处理数千并发连接                   |         |
|                    +----------------------------------------------+         |
|                                                                             |
|   单个 Worker 线程处理多连接示意：                                           |
|                                                                             |
|   +---------------------------------------------------------------------+  |
|   |                        Worker Thread                                 |  |
|   |                                                                     |  |
|   |   时间线 -------------------------------------------------------->  |  |
|   |                                                                     |  |
|   |   Connection 1: [读取] [挂起...] [恢复] [处理] [写入] [挂起] [恢复] |  |
|   |   Connection 2:      [读取] [挂起...] [恢复] [处理] [写入]          |  |
|   |   Connection 3:           [读取] [挂起...] [恢复] [处理]            |  |
|   |   Connection 4:                [读取] [挂起...] [恢复]              |  |
|   |                                                                     |  |
|   |   线程始终在忙碌，没有空闲等待！                                     |  |
|   |                                                                     |  |
|   +---------------------------------------------------------------------+  |
|                                                                             |
|   优点：                                                                     |
|   - 一个连接一个协程，比线程更轻量                                          |
|   - I/O 等待时协程挂起，线程不阻塞                                          |
|   - 单个 Worker 线程可处理数千并发连接                                      |
|   - 代码结构清晰，避免回调地狱                                              |
|                                                                             |
+-----------------------------------------------------------------------------+
```

## 4. 核心改进点 3: 线程安全的跨线程通信

### 错误做法

```
+-----------------------------------------------------------------------------+
|                        X 错误做法：直接跨线程操作                            |
+-----------------------------------------------------------------------------+
|                                                                             |
|   Main Thread                              Worker Thread                    |
|   +------------------+                     +------------------+            |
|   |                  |                     |                  |            |
|   |  accept 新连接   |                     |  epoll_wait()    |            |
|   |        |         |                     |        |         |            |
|   |        v         |                     |        v         |            |
|   |  epoll_ctl(ADD)  | ------------------> |   !! 数据竞争!!  |            |
|   |  直接操作 Worker |                     |   可能崩溃！     |            |
|   |  的 epoll        |                     |   数据损坏！     |            |
|   |                  |                     |                  |            |
|   +------------------+                     +------------------+            |
|                                                                             |
|   问题：多线程同时操作同一个 epoll_fd 会导致数据竞争                         |
|                                                                             |
+-----------------------------------------------------------------------------+
```

### 正确做法

```
+-----------------------------------------------------------------------------+
|                        OK 正确做法：队列 + eventfd 唤醒                      |
+-----------------------------------------------------------------------------+
|                                                                             |
|   Main Thread                              Worker Thread                    |
|   +------------------+                     +------------------+            |
|   |                  |                     |                  |            |
|   |  accept 新连接   |                     |  epoll_wait()    |            |
|   |        |         |                     |        |         |            |
|   |        v         |                     |        v         |            |
|   |  lock(mutex)     |                     |  返回 (wake_fd)  |            |
|   |        |         |                     |        |         |            |
|   |        v         |                     |        v         |            |
|   |  push to queue   |                     |  read(wake_fd)   |            |
|   |        |         |                     |        |         |            |
|   |        v         |                     |        v         |            |
|   |  unlock          |                     |  lock(mutex)     |            |
|   |        |         |                     |        |         |            |
|   |        v         |                     |        v         |            |
|   |  write(wake_fd,1)| -- eventfd 唤醒 --> |  取出连接        |            |
|   |        |         |                     |        |         |            |
|   |        v         |                     |        v         |            |
|   |  继续accept      |                     |  unlock          |            |
|   |                  |                     |        |         |            |
|   |                  |                     |        v         |            |
|   |                  |                     |  epoll_ctl(ADD)  |            |
|   |                  |                     |        |         |            |
|   |                  |                     |        v         |            |
|   |                  |                     |  启动协程        |            |
|   |                  |                     |                  |            |
|   +------------------+                     +------------------+            |
|                                                                             |
|   优点：                                                                     |
|   - 使用 mutex 保护连接队列                                                 |
|   - 使用 eventfd 实现跨线程唤醒                                             |
|   - Worker 线程自行接管连接，避免并发问题                                    |
|   - 符合 "one-loop-per-thread" 模式                                        |
|                                                                             |
+-----------------------------------------------------------------------------+
```

## 5. 协程句柄详解

```
+-----------------------------------------------------------------------------+
|                     协程句柄 (coroutine_handle) 详解                         |
+-----------------------------------------------------------------------------+
|                                                                             |
|   问题 1：read_waiter_ = handle;  // 这个 handle 是谁的？                   |
|   答案：handle 是当前正在挂起的协程的句柄，即 HandleConnectionCo() 协程      |
|                                                                             |
|   问题 2：如果不保存句柄，协程不会自动恢复吗？                                |
|   答案：协程不会自动恢复！必须显式调用 handle.resume() 才能恢复               |
|                                                                             |
|   关键理解：                                                                 |
|   ==========                                                                |
|   - 协程挂起后，控制权返回给 PollOnce()                                      |
|   - PollOnce() 继续处理其他事件                                              |
|   - 当 socket 可读时，必须知道要恢复哪个协程                                  |
|   - read_waiter_ 就是用来记录"哪个协程在等待这个 socket"                     |
|   - 如果不保存，协程会永远挂起（内存泄漏）                                    |
|                                                                             |
+-----------------------------------------------------------------------------+
|                                                                             |
|   协程层级关系：                                                             |
|                                                                             |
|   WorkerLoop::StartConnectionCoroutine()                                    |
|       |                                                                     |
|       v                                                                     |
|   HandleConnectionCo()  <--- 这就是 handle 指向的协程                       |
|       |                                                                     |
|       |  co_await ReadRequestCo()                                           |
|       |         |                                                           |
|       |         v                                                           |
|       |   ReadAwaiter::await_suspend(handle)                                |
|       |         |                                                           |
|       |         |  handle = HandleConnectionCo 协程的句柄                    |
|       |         |  read_waiter_ = handle  // 保存，等待后续恢复              |
|       |         |                                                           |
|       |         v                                                           |
|       |   协程挂起，控制权返回 PollOnce()                                   |
|       |                                                                     |
|   后续恢复：                                                                 |
|       |                                                                     |
|   NotifyReadable()                                                          |
|       |                                                                     |
|       v                                                                     |
|   ResumeWaiter(read_waiter_)  // 用保存的句柄恢复 HandleConnectionCo        |
|       |                                                                     |
|       v                                                                     |
|   HandleConnectionCo() 继续执行...                                          |
|                                                                             |
+-----------------------------------------------------------------------------+
```

### 完整的协程挂起/恢复流程

```
+-----------------------------------------------------------------------------+
|                    await_suspend(handle) 流程图                              |
+-----------------------------------------------------------------------------+
|                                                                             |
|   步骤 1: 协程执行 co_await                                                  |
|   ============================                                              |
|                                                                             |
|   HandleConnectionCo() 协程执行:                                            |
|       co_await connection->ReadRequestCo(&error);                           |
|                                                                             |
|                           |                                                 |
|                           v                                                 |
|   步骤 2: 创建 Awaiter 并调用 await_ready                                    |
|   =======================================                                   |
|                                                                             |
|   ReadRequestCo() 返回一个 ReadAwaiter 对象                                 |
|       await_ready() 检查数据是否就绪                                        |
|           - 返回 true: 不挂起，直接继续                                     |
|           - 返回 false: 需要等待，进入 await_suspend                        |
|                                                                             |
|                           |                                                 |
|                           v                                                 |
|   步骤 3: 调用 await_suspend(handle)                                        |
|   =================================                                         |
|                                                                             |
|   bool await_suspend(std::coroutine_handle<> handle) {                     |
|       // handle 指向 HandleConnectionCo 协程                                |
|       // 此时 HandleConnectionCo 正在挂起                                   |
|                                                                             |
|       connection_->read_waiter_ = handle;  // 保存协程句柄                  |
|       connection_->state_ = State::kReading;                               |
|       connection_->ArmReadDeadline();       // 设置超时                     |
|       return true;  // 返回 true 表示真正挂起                               |
|   }                                                                         |
|                                                                             |
|   注意：此时 HandleConnectionCo 协程被挂起，控制权返回给调用者               |
|                                                                             |
|                           |                                                 |
|                           v                                                 |
|   步骤 4: 协程挂起，PollOnce 继续运行                                        |
|   =======================================                                   |
|                                                                             |
|   PollOnce() 继续处理其他事件，线程不阻塞                                    |
|   其他连接的协程可以被调度执行                                               |
|                                                                             |
|                           |                                                 |
|                           v                                                 |
|   步骤 5: 数据到达，恢复协程                                                 |
|   ===============================                                           |
|                                                                             |
|   epoll_wait() 返回 EPOLLIN 事件                                            |
|       |                                                                     |
|       v                                                                     |
|   NotifyReadable() {                                                        |
|       ResumeWaiter(read_waiter_);  // 用保存的句柄恢复协程                  |
|   }                                                                         |
|       |                                                                     |
|       v                                                                     |
|   handle.resume()  // 恢复 HandleConnectionCo 协程                          |
|       |                                                                     |
|       v                                                                     |
|   await_resume() 被调用，返回读取结果                                       |
|       |                                                                     |
|       v                                                                     |
|   HandleConnectionCo() 继续执行...                                          |
|                                                                             |
+-----------------------------------------------------------------------------+
```

|       }                                                                     |
|   }                                                                         |

## 总结

本 RPC 框架通过以下核心改进实现了高性能、高并发：

1. **one-loop-per-thread**：主线程负责 accept，Worker 线程驱动事件循环
2. **协程驱动 I/O**：每连接一协程，非阻塞高效处理
3. **线程安全通信**：queue + eventfd 实现安全的跨线程分发
4. **超时管理**：Tick 检查 + deadline 机制确保资源释放
5. **RAII 资源管理**：UniqueFd 自动管理 fd 生命周期

这些改进使得框架能够：
- 在单台机器上支持数万并发连接
- 充分利用多核 CPU 资源
- 保持代码结构清晰、易于维护
