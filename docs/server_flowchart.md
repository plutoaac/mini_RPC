# RPC Server 流程图

本文档详细描述了 RPC 服务器的架构和运行流程。

## 1. 整体架构图

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                                   RPC Server                                     │
│                                                                                  │
│  ┌────────────────────────────────────────────────────────────────────────────┐ │

│  │                                                                             │ │
│  │   ┌─────────────┐    ┌─────────────┐    ┌───────────────────────────────┐  │ │
│  │   │ listen_fd   │    │ epoll_fd    │    │     connections (map)         │  │ │
│  │   │ (监听socket)│    │ (epoll实例) │    │  fd -> Connection             │  │ │
│  │   └─────────────┘    └─────────────┘    └───────────────────────────────┘  │ │
│  └────────────────────────────────────────────────────────────────────────────┘ │
│                                                                                  │
│                              epoll 事件循环                                      │
│                                                                                  │
│              EPOLLIN  ──────> 新连接到达 / 数据可读                              │
│              EPOLLOUT ──────> 可以发送数据                                       │

```

## 3. epoll 事件循环

```
┌──────────────────────────────────────────────────────────────────────────────────┐
│                              主事件循环 (无限循环)                                 │
│                                                                                   │
│                epoll_wait(events, 64, timeout=-1)                                │
│                      等待事件发生 (阻塞)                                          │
│                                                                                   │
│                             ┌────────┴────────┐                                   │
│                             │  事件就绪?      │                                   │
│                             └────────┬────────┘                                   │
│                           是 │             │ 否 (错误)                             │
│                              ▼             ▼                                       │
│                    ┌──────────────┐  ┌──────────────────┐                         │
│                    │ 遍历 events  │  │ EINTR: continue  │                         │
│                    └──────┬───────┘  │ 其他: return     │                         │
│                           │          └──────────────────┘                         │
│                           ▼                                                        │
│   ┌───────────────────────────────────────────────────────────────────────────┐  │
│   │                         事件类型判断                                       │  │
│   │                                                                           │  │
│   │        fd == listen_fd ?                                                  │  │
│   │      ┌─────┴─────┐                                                        │  │
│   │      ▼           ▼                                                        │  │
│   │  ┌───────┐   ┌────────────────────────────────────────┐                    │  │
│   │  │新连接 │   │          客户端 socket 事件             │                    │  │
│   │  │处理   │   │                                        │                    │  │
│   │  └───┬───┘   │  EPOLLIN  ──> OnReadable()            │                    │  │
│   │      │       │  EPOLLOUT ──> OnWritable()            │                    │  │
│   │      │       │  EPOLLERR/HUP/RDHUP ──> MarkClosing() │                    │  │
│   │      │       │  更新 epoll 事件 (EPOLL_CTL_MOD)       │                    │  │
│   │      │       └────────────────────────────────────────┘                    │  │
│   │      ▼                                                                      │  │
│   │  ┌──────────────────────────────────────────────────────────────────┐      │  │
│   │  │                    Accept 循环 (非阻塞)                           │      │  │
│   │  │                                                                  │      │  │
│   │  │  while (true) {                                                  │      │  │
│   │  │      client_fd = accept4(SOCK_NONBLOCK | SOCK_CLOEXEC)           │      │  │
│   │  │      if (EAGAIN) break;   // 没有更多连接                        │      │  │
│   │  │      epoll_ctl(EPOLL_CTL_ADD, client_fd, EPOLLIN|EPOLLRDHUP)    │      │  │
│   │  │      connections.emplace(client_fd, Connection(fd, registry))   │      │  │
│   │  │  }                                                               │      │  │
│   │  └──────────────────────────────────────────────────────────────────┘      │  │
│   └───────────────────────────────────────────────────────────────────────────┘  │
│                                                                                   │
│                              (循环继续，等待下一个事件)                            │
└───────────────────────────────────────────────────────────────────────────────────┘
```

## 9. 核心组件职责

| 组件                | 职责                                   | 关键方法                        |
| ------------------- | -------------------------------------- | ------------------------------- |
| **RpcServer**       | 服务器生命周期管理，epoll 事件循环     | `Start()`                       |
| **Connection**      | 单个客户端连接处理，协议解析，请求路由 | `OnReadable()`, `OnWritable()`  |
| **ServiceRegistry** | 服务方法注册与查找，线程安全           | `Register()`, `Find()`          |

## 10. 设计特点

1. **单线程 epoll 事件驱动**: 高效处理多连接，无需多线程
2. **非阻塞 I/O**: 所有 socket 操作都是非阻塞的
3. **RAII 资源管理**: `UniqueFd` 自动管理 socket 生命周期
4. **简洁的帧协议**: 4字节大端序长度前缀 + Protobuf 序列化
5. **线程安全注册表**: `ServiceRegistry` 支持并发注册和查找
6. **动态事件注册**: 根据写缓冲区状态动态调整 EPOLLOUT 事件
